## 서비스 소개 페이지 모바일 UX 개선

### 문제
모바일에서 탭이 페이지 최상단에만 있어, 콘텐츠를 끝까지 읽은 뒤 다른 솔루션으로 이동하려면 상단까지 다시 스크롤해야 하는 불편함 존재.

### 해결: 모바일 전용 아코디언 UI 도입

#### 구조 분리
- `lg:hidden` / `hidden lg:flex` 로 모바일·데스크탑 UI를 완전히 분리
- 데스크탑: 기존 좌측 탭 + 우측 콘텐츠 그대로 유지
- 모바일: 탭 버튼 제거, 아코디언으로 전환

#### 아코디언 동작
- 초기 상태: 3개 모두 접힘 (`openIdx: null`) — 1/2/3 순서 편향 방지
- 하나만 열리는 방식 (토글: 열린 항목 재클릭 시 닫힘)
- `openIdx === idx ? null : idx` 토글 로직

#### 타이틀 중복 제거
- 기존: 탭 선택 UI(레이블) + 콘텐츠 상단 타이틀 카드가 동일 정보를 두 번 노출
- 해결: 아코디언 헤더가 열릴 때 타이틀 카드 역할로 확장
  - **접힘**: `[01] AI 스마트 면접 설계` compact 레이블 + chevron
  - **열림**: accent 배경 + num/tag/title/subtitle 풀 표시 → 별도 타이틀 카드 불필요
- 본문(`SolutionBody`)은 데모 iframe + 피처 리스트만 렌더링

#### SolutionBody 컴포넌트 추출
- 데모 iframe + 피처 카드 블록을 `SolutionBody` 컴포넌트로 분리
- 모바일 아코디언 본문 / 데스크탑 우측 영역에서 동일하게 재사용
- 코드 중복 제거 및 향후 솔루션 추가·변경 시 단일 수정 지점 확보

#### 아코디언 스타일 (Shadow Elevation)
- **접힘**: `border border-slate-100` + `shadow-sm`, 흰 배경 — flat 카드
- **열림**: border 완전 제거, 중립 box-shadow(`rgba(0,0,0,0.13)`)만으로 카드 부각
  - accent 색 border/glow 제거 → 헤더 배경색만으로 활성 상태 표현, 더 정돈된 느낌

#### iframe 최적화
- 아코디언 열릴 때만 `DemoFrame` 마운트 (조건부 렌더링)
- 페이지 로드 시 iframe 3개 동시 요청 방지

#### 확장성
- `SOLUTIONS` 배열 수정만으로 솔루션 추가·삭제·순서 변경 가능
- 인덱스 하드코딩 없이 전부 `.map()` 기반, UI 자동 반영
